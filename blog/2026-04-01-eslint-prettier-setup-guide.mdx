## 📚 为什么要安装这两个工具？

在开始之前，先简单了解一下这两位"代码守护者"：

### ESLint - 代码质量检查员

**职责：** 检查代码中的问题和潜在 bug

**能发现什么：**
- ❌ 未使用的变量
- ❌ 语法错误
- ❌ React Hooks 使用不当
- ❌ 不符合最佳实践的写法

**示例：**
```javascript
// ❌ 会被 ESLint 标记
var count = 0;  // 应该用 const/let
console.log(x); // x 未定义

// ✅ 正确的写法
const count = 0;
```

### Prettier - 代码格式化专家

**职责：** 统一代码格式，让所有代码看起来像一个人写的

**能做什么：**
- 📏 自动调整缩进（2 空格）
- 💬 统一引号风格（单引号优先）
- ⏎ 控制行尾分号（必须加分号）
- 📝 处理换行和空格

**对比效果：**

```javascript
// 格式化前乱七八糟
function test( ){console.log( "hello" )}

// 格式化后整整齐齐
function test() {
  console.log('hello');
}
```

### 为什么需要两个工具一起用？

很多新手会问："一个工具不够吗？"

**答案是：不够！** 🙅‍♂️

- **ESLint** 专注于代码**质量**（有没有 bug、是否符合规范）
- **Prettier** 专注于代码**样式**（好不好看、是否统一）

就像做饭一样：
- ESLint 是食品安全检查员（确保菜没坏、能吃）
- Prettier 是摆盘厨师（确保菜品好看、整齐）

两者配合，才能做出既安全又美味的菜肴！🍳

## 🛠️ 第一步：安装依赖

打开终端，在项目根目录执行：

```bash
# 安装 ESLint 及相关插件
yarn add --dev \
  eslint \
  @docusaurus/eslint-plugin \
  eslint-plugin-react \
  eslint-plugin-react-hooks \
  eslint-config-prettier \
  prettier
```

**依赖说明：**
- `eslint` - 核心工具
- `@docusaurus/eslint-plugin` - Docusaurus 专用规则
- `eslint-plugin-react` - React 规则
- `eslint-plugin-react-hooks` - React Hooks 规则
- `eslint-config-prettier` - 避免与 Prettier 冲突
- `prettier` - 格式化工具

安装完成后，你会看到 `package.json` 的 `devDependencies` 中多了这些依赖。

## 📝 第二步：创建配置文件

### 1. 创建 ESLint 配置（Flat Config 格式）

在项目根目录创建 `eslint.config.js`：

```javascript
// @ts-check

const js = require('@eslint/js');
const globals = require('globals');
const reactHooksPlugin = require('eslint-plugin-react-hooks');
const prettierConfig = require('eslint-config-prettier');

/** @type {import('eslint').Linter.Config[]} */
module.exports = [
  // 全局忽略的文件
  {
    ignores: [
      'build/',
      '.docusaurus/',
      'node_modules/',
      '*.min.js',
      '*.bundle.js',
      'logs/',
      '*.log',
    ],
  },

  // 基础配置
  {
    files: ['**/*.{js,jsx,ts,tsx}'],
    languageOptions: {
      ecmaVersion: 'latest',
      sourceType: 'module',
      globals: {
        ...globals.browser,
        ...globals.node,
      },
      parserOptions: {
        ecmaFeatures: {
          jsx: true,
        },
      },
    },
    plugins: {
      'react-hooks': reactHooksPlugin,
    },
  },

  // 推荐规则
  js.configs.recommended,
  
  // React Hooks 规则
  {
    rules: {
      ...reactHooksPlugin.configs.recommended.rules,
    },
  },
  
  // React 规则（简化版）
  {
    files: ['**/*.{js,jsx,ts,tsx}'],
    rules: {
      // React 基本规则
      'react/react-in-jsx-scope': 'off', // React 17+ 不需要
      
      // 代码质量相关
      'no-console': 'warn', // 警告 console.log
      'no-unused-vars': ['warn', { vars: 'all', args: 'none' }], // 未使用的变量
      'prefer-const': 'error', // 能用 const 就不用 let
      'no-var': 'error', // 禁止使用 var
      
      // 代码风格（大部分交给 Prettier 处理）
      'quotes': ['warn', 'single', { 
        'avoidEscape': true,
        'allowTemplateLiterals': true
      }],
      'semi': ['error', 'always'], // 必须分号
      'indent': 'off', // 缩进交给 Prettier
      'comma-dangle': ['warn', 'only-multiline'], // 多行时允许尾随逗号
    },
  },

  // Prettier 配置（必须放在最后）
  prettierConfig,
];
```

**⚠️ 注意：** ESLint v10+ 使用新的 Flat Config 格式，不再是 `.eslintrc.js`！

### 2. 创建 Prettier 配置

在项目根目录创建 `.prettierrc`：

```json
{
  "printWidth": 100,
  "tabWidth": 2,
  "useTabs": false,
  "semi": true,
  "singleQuote": true,
  "quoteProps": "as-needed",
  "jsxSingleQuote": false,
  "trailingComma": "all",
  "bracketSpacing": true,
  "bracketSameLine": false,
  "arrowParens": "always",
  "endOfLine": "lf",
  "proseWrap": "preserve",
  "overrides": [
    {
      "files": "*.md",
      "options": {
        "proseWrap": "always",
        "printWidth": 120
      }
    },
    {
      "files": "*.mdx",
      "options": {
        "proseWrap": "preserve"
      }
    }
  ]
}
```

**配置解读：**
- `printWidth: 100` - 每行最多 100 个字符
- `tabWidth: 2` - 2 空格缩进
- `singleQuote: true` - 优先使用单引号
- `semi: true` - 语句末尾必须加分号
- `trailingComma: all` - 多行时对象/数组最后一项添加逗号

```json
// 保存时自动修复 ESLint 问题

"editor.defaultFormatter": "esbenp.prettier-vscode",

// 保存时自动格式化
"editor.formatOnSave": true
```

**✨ 这就是自动格式化的核心配置！**

## 🚀 使用方法

配置完成后，你就可以享受自动化带来的便利了！

### 方式 1：保存时自动执行（最推荐）

这是最常用的方式，只需：

```
编写代码 → 按 Ctrl+S 保存 → 自动格式化 + 修复
```

**实际效果：**

假设你写了这样的代码：

```javascript
// 保存前：乱七八糟
var unused="test";
function test( ){console.log( "hello" )}
```

按 `Ctrl+S` 保存后，自动变成：

```javascript
// 保存后：整整齐齐
const unused = 'test'; // ESLint 会警告这个变量未使用
function test() {
  console.log('hello');
}
```

**是不是很神奇？** ✨

### 方式 2：手动触发格式化

如果忘记保存，可以手动触发：

**快捷键：**
- Windows: `Shift + Alt + F`
- Mac: `Shift + Option + F`

**菜单操作：**
```
按 F1 → 输入 "Format Document" → 回车
```

### 方式 3：命令行运行 ESLint

在终端中手动检查：

```bash
# 检查整个项目
npx eslint .

# 检查并自动修复
npx eslint . --fix

# 检查特定文件
npx eslint src/components/*.js
```

**输出示例：**
```
D:\Docusaurus\test-site\src\components\ForMDX.js
  1:8  warning  'React' is defined but never used     no-unused-vars
  3:8  warning  'BrowserOnly' is defined but never used  no-unused-vars

✖ 2 problems (0 errors, 2 warnings)
```

## 🔧 常见问题解决

### 问题 1：保存时没有自动格式化

**检查清单：**

1. ✅ VS Code 是否安装了 Prettier 扩展
2. ✅ `.vscode/settings.json` 中是否有以下配置：
   ```json
   "editor.formatOnSave": true,
   "editor.defaultFormatter": "esbenp.prettier-vscode"
   ```
3. ✅ 重启 VS Code：`Ctrl+Shift+P` → `Developer: Reload Window`

### 问题 2：ESLint 没有报错

**可能原因：**

1. VS Code 没有安装 ESLint 扩展
2. 配置文件路径不对
3. ESLint 服务没有启动

**解决方法：**

```bash
# 测试 ESLint 是否正常工作
npx eslint --version

# 如果报错，重新安装
yarn add --dev eslint
```

### 问题 3：MDX 文件不生效

ESLint 默认不支持 MDX，需要额外配置。

**解决方案 1：** 安装 MDX 插件

```bash
yarn add --dev eslint-plugin-mdx
```

然后在 `eslint.config.js` 中添加 MDX 支持。

**解决方案 2：** 在 VS Code 设置中添加 MDX 验证

```json
"eslint.validate": [
  "javascript",
  "javascriptreact",
  "typescript",
  "typescriptreact",
  "mdx"
]
```

### 问题 4：ESLint 和 Prettier 规则冲突

**不用担心！** 我们的配置已经解决了这个问题。

在 `eslint.config.js` 的最后添加了：

```javascript
// Prettier 配置（必须放在最后）
prettierConfig,
```

这会自动关闭所有与 Prettier 冲突的 ESLint 规则。

## 📊 性能优化建议

### 如果感觉 linting 太慢

在 `.vscode/settings.json` 中调整时间预算：

```json
{
  "eslint.timeBudget": {
    "onValidation": 3000,  // 增加到 3 秒
    "onFixes": 3000
  }
}
```

### 如果只想检查特定目录

在 `eslint.config.js` 的 `ignores` 中排除不需要的目录：

```javascript
{
  ignores: [
    'build/',
    '.docusaurus/',
    'node_modules/',
    'old-code/',  // 不检查旧代码
  ],
}
```
